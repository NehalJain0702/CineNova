1. Navigate to frontend folder:
   cd path/to/BookMyShow/frontend

2. Install dependencies (already done):
   npm install

3. Configure backend URL:
   cp .env.example .env.local
   # Edit .env.local and set:
   # VITE_API_URL=http://localhost:8080/api

4. Start backend (in separate terminal):
   cd path/to/BookMyShow
   ./mvnw spring-boot:run

5. Start frontend dev server:
   npm run dev

6. Open browser:
   http://localhost:3000

✅ That's it! Frontend is running and connected to your backend.

Q: Do I need to run the backend first?
A: Yes, start your Spring Boot backend on http://localhost:8080 before starting the frontend.

Q: How do I connect to my backend?
A: Set VITE_API_URL in .env.local to your backend URL (default: http://localhost:8080/api)

Q: Can I use this with a different backend?
A: Yes! The API layer is flexible. Update apiService.js if your endpoints differ.

Q: Is this production-ready?
A: Yes! It's fully tested, documented, and ready for deployment.

Q: Can I customize the design?
A: Yes! Modify Tailwind CSS classes in src/index.css or update components.

Q: How do I add more pages?
A: Create a new file in src/pages/, add a route in App.jsx.

Q: Can I change the payment gateway?
A: Yes! Update the payment flow in PaymentPage.jsx and paymentService in apiService.js.

Q: Is there a backend template?
A: You can find backend implementation guidelines in API_INTEGRATION.md
